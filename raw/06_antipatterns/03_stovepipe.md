# 안티패턴: Stovepipe (Silo) System

## 정의
서로 통합되지 않은, 독립적으로 구축된 시스템/모듈들의 집합.
각 시스템은 자신의 특정 기능만을 위해 만들어졌고, 다른 시스템과
데이터를 공유하거나 연계되지 않는다.

## 발생 원인
- 기능을 추가할 때마다 기존 구조와의 통합을 고려하지 않고 "새로 하나 더" 만드는 방식으로 개발
- 데이터 모델/저장 방식에 대한 전체적인 설계 없이 기능별로 독립적인 저장소를 사용

## 문제점
- 동일하거나 유사한 데이터가 여러 곳에 중복 저장되어 일관성 문제 발생
- 전체 데이터를 가로질러 분석/조회하기 어려움 (예: 사용자 행동을 종합적으로 보기 어려움)
- 동일한 기능(예: 인증, 로깅)이 여러 곳에 중복 구현됨

## 회피 방법
- 새 기능을 추가하기 전, 기존 데이터 모델/모듈과의 연계 지점을 먼저 검토
- 공통 기능(인증, 로깅, 사용자 정보 등)은 공유 모듈/서비스로 통합
- 데이터 흐름도(00_decision_framework/02_architecture_style_decision_guide.md의
  "데이터 흐름 설계 체크리스트")를 통해 전체 데이터의 단일 진실 공급원(Single Source of Truth)을 정의

## agent 설계 시 적용
- 새로운 기능/모듈을 설계할 때, 기존에 정의된 데이터 모델과 어떻게 연계되는지를
  반드시 함께 명시한다. "독립적으로 새 저장소를 만든다"는 결정은 ADR로
  명확한 근거와 함께 남겨야 한다.
