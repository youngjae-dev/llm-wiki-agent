# C4 모델: 아키텍처 시각화 가이드

## 핵심 개념
C4 모델은 소프트웨어 아키텍처를 4개의 추상화 레벨(Context, Container, Component, Code)로
나누어 표현하는 방법론이다. 지도(Map)를 보듯 "전체 → 점점 확대"하는 방식으로,
보는 사람의 관심 수준에 맞는 다이어그램을 제공한다.

agent는 설계 결과를 문서화할 때 최소한 Context와 Container 레벨까지는
다이어그램(또는 다이어그램 코드, 예: Mermaid/PlantUML)으로 표현해야 한다.

## 레벨 1: System Context Diagram
- 가장 추상화된 수준. "이 시스템"을 하나의 박스로 표현하고,
  이 시스템과 상호작용하는 사용자(Actor)와 외부 시스템들을 함께 그린다.
- 목적: 시스템의 범위와 외부 의존성을 한눈에 파악

예시 구성요소:
- Person: 사용자 (예: "일반 사용자", "관리자")
- Software System (이 프로젝트): 박스 하나
- External Systems: 외부 API, 결제 시스템, 이메일 발송 서비스 등

## 레벨 2: Container Diagram
- 시스템을 구성하는 "배포 가능한 단위(컨테이너)"들로 확대
- 컨테이너 예시: 웹 애플리케이션(프론트엔드), API 서버(백엔드), 데이터베이스,
  메시지 큐, 캐시 서버 등
- 각 컨테이너 간의 통신 방식(REST, 비동기 메시지 등)을 화살표와 레이블로 표시
- 목적: 전체 시스템의 기술 구성과 데이터 흐름의 큰 그림을 파악

## 레벨 3: Component Diagram
- 특정 컨테이너(예: API 서버) 내부를 구성하는 주요 컴포넌트(모듈)들로 확대
- 예: 인증 모듈, 주문 처리 모듈, 알림 모듈 등과 이들 간의 관계
- 목적: 해당 컨테이너 내부 구조 이해 (개인 프로젝트에서는 모듈형 모놀리식의
  각 도메인 모듈을 여기에 대응시키면 됨)

## 레벨 4: Code Diagram
- 클래스 다이어그램 수준의 세부 구현 (선택 사항)
- 일반적으로 IDE의 자동 생성 기능으로 대체 가능하므로, 문서로 직접 관리할 필요는 낮음

## 표기 도구
- Mermaid: 마크다운에 바로 삽입 가능한 다이어그램 문법 (GitHub 등에서 렌더링됨)
- PlantUML + C4-PlantUML: C4 모델 전용 매크로를 제공하여 일관된 표기 가능
- Structurizr: C4 모델 전용 모델링 도구 (DSL 기반)

## agent 출력 규칙
1. 설계 결과를 정리할 때 System Context Diagram으로 시작 (이 프로젝트와 외부 의존성)
2. Container Diagram으로 내부 구성요소(서버, DB, 캐시, 큐 등)와 데이터 흐름을 표현
3. 필요시 핵심 모듈에 대한 Component Diagram 추가
4. 다이어그램은 Mermaid 코드 블록으로 출력하여 마크다운 문서에 바로 포함되도록 함
