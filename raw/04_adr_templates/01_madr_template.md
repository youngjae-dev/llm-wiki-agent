# ADR 템플릿 (MADR 스타일)

설계 결정을 내릴 때마다 아래 형식으로 문서를 작성한다.
출처: Markdown Any Decision Records(MADR) 프로젝트의 템플릿 구조를 참고하여 재구성.

## 제목
- 짧고 명령문 형태로 작성 (예: "사용자 인증 방식으로 JWT 채택")

## 메타데이터
- 결정 참여자: (1인 개발이면 본인)
- 날짜: YYYY-MM-DD
- 상태: 제안됨 / 채택됨 / 폐기됨 / 대체됨

## 컨텍스트와 문제 정의
- 어떤 문제/질문을 해결하기 위한 결정인지 2~3문장으로 설명
- 가능하면 질문 형태로 표현 ("어떤 방식으로 사용자 인증을 처리해야 하는가?")

## 결정 기준 (Decision Drivers)
- 이 결정에 영향을 주는 요구사항/제약 나열
  - 예: 무료 인프라 사용, 모바일 앱과의 호환성, 향후 소셜 로그인 확장 가능성

## 고려한 옵션들
- 옵션 1: ...
- 옵션 2: ...
- 옵션 3: ...

## 결정 결과
- 선택한 옵션: "옵션 N"
- 선택 이유: 다른 옵션과 비교했을 때 어떤 결정 기준을 가장 잘 만족시키는지 설명

## 옵션별 장단점 비교 (선택)
### 옵션 1: ...
- 장점: ...
- 단점: ...

### 옵션 2: ...
- 장점: ...
- 단점: ...

## 결과 / 영향
- 이 결정으로 인해 추가로 필요한 작업, 향후 재검토가 필요한 시점 등을 기록
